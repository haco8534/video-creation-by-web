# ⏱️ タイムスタンプ自動更新ワークフロー

## 概要

Phase B で生成された `scene_durations.json` の累積開始時刻をもとに、
完成品フォルダ内の `概要欄.md` に含まれるタイムスタンプ（推定値）を
実際の動画時間に書き換える。

### インプット
- `presentation/{project_id}/scene_durations.json` — Phase B で自動生成
- `output/ready/{テーマ名}/概要欄.md` — Phase A の Step 5b で生成

### アウトプット
- `output/ready/{テーマ名}/概要欄.md` — タイムスタンプ行が正確な `MM:SS` に書き換えられた状態

---

## 前提条件

- Phase B（メインコンテンツ生成）が完了しており、`scene_durations.json` が存在すること
- `概要欄.md` にタイムスタンプ行（`MM:SS セクション名` 形式）が含まれていること

---

## 実行手順

### Step 1: タイムスタンプ更新スクリプトの実行

// turbo
```powershell
node tools/update_timestamps.js {project_id} "{完成品DIR}/概要欄.md"
```

**作業ディレクトリ**: `d:\myfolder\動画生成\ふぁくとラボ\presentation`

**引数**:
- `{project_id}` — プレゼンテーションのプロジェクトID（例: `money_happiness`）
- `{完成品DIR}/概要欄.md` — 書き換え対象の概要欄ファイルパス

### Step 2: 結果の確認

スクリプトの出力ログで以下を確認する：

- [ ] すべてのタイムスタンプ行にシーンが割り当てられたか（❌ が出ていないか）
- [ ] 割り当てが時系列順（時間が後ろに進む方向）になっているか
- [ ] 「推定値」コメントが除去されたか

### Step 3: 手動微調整（必要な場合のみ）

スクリプトは scene_durations のシーンタイトルと概要欄ラベルの部分一致で対応を推定する。
以下の場合は手動で微調整する：

1. **Block 先頭のタイトルカードシーンではなく、Block 内の別シーンにマッチした場合**
   - 正しいBlock 先頭シーンのタイムスタンプに手動で変更する
   - 確認方法: `scene_durations.json` を開き、各Blockの先頭シーンIDの累積時間を確認

2. **ラベルとシーンタイトルの一致度が低く、間違ったシーンにマッチした場合**
   - ログ出力の `score` 値が 3（最低マッチ）の行は特に注意して確認する

---

## マッチングの仕組み

スクリプトは以下の戦略でタイムスタンプ行とシーンを対応付ける：

1. 概要欄のラベルを `—` `──` `-` `:` 等で分割してキーワードを抽出
2. 各キーワードと scene_durations の各シーンタイトルの**3文字以上の共通部分文字列**を検索
3. 最長一致文字数をスコアとして使用
4. **時系列制約**: 前に割り当てたシーンより後のシーンのみ候補にする
5. **予約制約**: 後続タイムスタンプ行のために十分なシーンを残す
6. `00:00` の行は必ず scene 0 を割り当て

---

## トラブルシューティング

| 問題 | 対処法 |
|------|--------|
| `scene_durations.json not found` | Phase B を実行して音声生成を完了させる |
| `No timestamp lines found` | 概要欄に `MM:SS セクション名` 形式の行があるか確認 |
| タイムスタンプが実際の動画とずれている | scene_durations.json の時間は音声生成時点の値。レンダリング後の動画時間と完全一致するはず |
| 間違ったシーンにマッチした | Step 3 の手動微調整を行う |
