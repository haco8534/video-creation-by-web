# 🔍 サムネイルプロンプト推敲ワークフロー

## 概要

`thumnail\output\` に保存済みのサムネイルプロンプトファイルを、`channel_thumbnail_guide.md` の原則に基づいてレビュー・修正する。

### インプット
- **対象ファイル**: `thumnail\output\prompts.txt`（全プロンプト一括ファイル）
- ユーザーがタイトル名を指定した場合は、そのタイトルのブロックのみを対象とする

### アウトプット
- **修正済みファイル**: 問題点を修正して上書き保存
- **推敲レポート**: 各ファイルの問題点と修正内容のサマリー

---

## 事前準備（必須参照）

推敲時は、必ず以下のガイドを参照すること：

**📄 `d:\myfolder\動画生成\thumnail\channel_thumbnail_guide.md`**

---

## 実行手順

### Step 1: ガイドの読み込み

`channel_thumbnail_guide.md` を読み込み、チェック基準を把握する。

### Step 2: 対象ファイルの特定

ユーザーの指示に応じて対象を決める。

```
# 指定方法
- 「全部推敲して」 → output/ 内の全ファイル
- 「デトックスを推敲して」 → デトックス.txt のみ
- ファイル名を複数指定 → 指定されたファイルのみ
```

### Step 3: 各ファイルのチェック

対象ファイルを1つずつ読み込み、以下の **7つの基準** でチェックする。

#### チェック項目一覧

| # | チェック項目 | 基準 | 重要度 |
|---|---|---|---|
| 1 | メインキャッチの文字数 | **8〜15文字**（厳守） | 🔴 必須 |
| 2 | サブテキストの文字数 | **5〜15文字** | 🔴 必須 |
| 3 | 方向性の重複 | 同一ファイル内で**異なる方向性**を使用 | 🔴 必須 |
| 4 | 誇張の有無 | タイトルから読み取れない誇張がないか | 🔴 必須 |
| 5 | ビジュアルの焦点 | **1つの主題**に絞れているか（キャラ多すぎ・背景凝りすぎはNG） | 🟡 重要 |
| 6 | 禁止表現チェック | ガイドの「やらないこと」に抵触していないか | 🟡 重要 |
| 7 | テーマ想起性 | サムネを見ただけで動画タイトルが想起できるか | 🟡 重要 |

#### 詳細チェック基準

##### 1. メインキャッチの文字数（🔴 必須）
- 8文字未満 → **短すぎ。もう少し情報を追加**
- 16文字以上 → **長すぎ。圧縮して8〜15文字に収める**
- 句読点・記号はカウントに含める
- 「？」「！」もカウント対象

##### 2. サブテキストの文字数（🔴 必須）
- 4文字以下 → **短すぎ。補足情報を追加**
- 16文字以上 → **長すぎ。5〜15文字に収める**
- サブテキストが省略されている場合はOK（任意項目）

##### 3. 方向性の重複（🔴 必須）
- 同一ファイル内の全デザイン案で `direction` がすべて異なること
- 重複がある場合 → **別の方向性に変更し、mood/visual_concept/styleも全面刷新**
- 使用可能な7方向性: 衝突・破壊 / 静寂の威圧 / 二極の落差 / 皮肉・風刺 / 問いかけ・不穏 / 教科書の裏切り / 証拠の提示

##### 4. 誇張の有無（🔴 必須）
- タイトルに書いていない内容を盛っていないか
- NG表現例: 「全部ウソ」「衝撃の真実」「ヤバすぎる」「まだ信じてるの？」
- メインキャッチとサブテキストの両方をチェック
- mood や visual_concept に誇張がないかも確認

##### 5. ビジュアルの焦点（🟡 重要）
- `visual_concept` が **1つの強い主題** に絞られているか
- キャラが3体以上 → **多すぎ。1〜2体に削減**
- 背景の説明が長すぎる → **シンプルに（単色、グラデ、軽いパターン）**
- 複数の要素が同時に主張している → **主役を1つに絞る**

##### 6. 禁止表現チェック（🟡 重要）
- `style` に以下が含まれていないか:
  - 写実・フォトリアル
  - AI風の厚塗り・重いレンダリング
  - カートゥーン調
  - AIっぽいキラキラ（パーティクル、光の粒子、グロー）
  - 書き込みすぎ・ディテール過多
- `visual_concept` に以下が含まれていないか:
  - 複雑な風景・室内シーン
  - テキストよりイラストが目立つ構図
  - 台本の具体的内容（研究者名、データ、数値、結論）

##### 7. テーマ想起性（🟡 重要）
- メインキャッチ + ビジュアルの組み合わせで、動画の内容が直感的にわかるか
- タイトルを知らない人が見ても「何について語っている動画か」が伝わるか
- メインキャッチだけで意味が通るか（サブテキストがなくても最低限伝わるか）

### Step 4: 修正の実施

問題が見つかった場合、以下のルールで修正する。

#### 修正ルール

1. **文字数違反** → 同じ意味を保ちつつ文字数を調整。意味が変わりすぎる場合は別の切り口に変更
2. **方向性の重複** → 重複している案を丸ごと刷新（direction, mood, visual_concept, style すべて書き換え）
3. **誇張** → タイトルから自然に読み取れる範囲の表現に書き換え
4. **ビジュアルの焦点ぼけ** → 要素を削って主題を1つに絞る
5. **禁止表現** → ガイドの推奨スタイルに沿って書き換え
6. **テーマ想起性が低い** → メインキャッチを動画の核心に寄せる

#### 修正しないもの

- 主観的な「もっと良い案がありそう」は推敲では行わない
- visual_concept の創造性・ユニークさは尊重する（ガイド違反でなければOK）
- 方向性の好みは変えない（チャンネルガイドに沿っていればOK）

### Step 5: ファイルの上書き保存

修正が完了したら、元のファイルを上書き保存する。

- 修正なしの場合はスキップ（ファイルを触らない）
- 修正ありの場合は修正後の内容で上書き

### Step 6: 推敲レポートの出力

すべてのファイルのチェック結果をレポートとして出力する。

#### レポートフォーマット

```
## 推敲結果サマリー

### ✅ 問題なし
- ファイル名.txt

### ⚠️ 修正あり
- ファイル名.txt
  - [チェック項目]: 修正前 → 修正後（修正理由）

### 📊 統計
- チェック対象: N ファイル
- 問題なし: N ファイル
- 修正あり: N ファイル
- 修正箇所合計: N 箇所
```

---

## ⚠️ 重要な注意事項

1. **推敲は機械的に行う**。ガイドの基準に照らして客観的に判断
2. **文字数は厳密にカウント**する（全角1文字、半角1文字としてカウント）
3. **修正は最小限**に。問題がなければ触らない
4. **方向性の重複は即修正**。これは絶対にNG
5. **誇張チェックは慎重に**。タイトルの内容を超えているかどうかが判断基準
6. **推敲後のファイルもChrome拡張にそのままコピペ可能**な形式を維持すること
