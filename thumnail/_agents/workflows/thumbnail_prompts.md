# 🎨 サムネイルプロンプト一括生成ワークフロー

## 概要

台本（1本 or 複数本）を入力として、YouTube動画サムネイル用の画像生成プロンプトを一括生成する。

### インプット
- **台本ファイル**（1本 or 複数本）: `台本作成/{テーマ名}/script.md`

### アウトプット
- **コードブロック1つ**に、1行1プロンプトでまとめた最終出力
- この出力をそのまま **Gemini Image Auto Generator 拡張機能** のテキスト入力欄にコピペして使う

### 拡張機能との連携

**Gemini Image Auto Generator**（Chrome拡張）は以下の仕様：
- テキスト入力欄に **1行 = 1プロンプト** で入力する
- 「まとめてキューに追加」→「スタート」で Gemini に順次自動送信される
- **したがって、本ワークフローの出力は「1行1プロンプト」のコードブロック**である

> ⚠️ プロンプト内に改行を入れてはいけない。1つのプロンプトは必ず1行に収める。
> ⚠️ CSVファイルへの保存は不要。チャット欄にコードブロックとして出力するだけでよい。
> ⚠️ 個別の案（案A、案B…）の見出しやマークダウン装飾はコードブロックの外に書く。コードブロック内はプロンプトのプレーンテキストのみ。

## デザインリファレンス（必須参照）

プロンプト生成時は、必ず以下のガイドを参照すること：

**📄 `d:\myfolder\動画生成\thumnail\channel_thumbnail_guide.md`**

このガイドに記載されている以下の項目を厳守する：
- チャンネルの方向性・視聴者層
- デザインの大原則（「事実がドラマチックなら、ドラマチックに出す」）
- スタイルの ✅やること / ❌やらないこと
- カラー規定（赤文字+白太縁取り、暗い背景、等）
- テキストのルール（メインキャッチ8〜15文字、サブテキスト5〜15文字）
- レイアウトパターン（A〜E型）
- キャッチコピーのパターン（直球否定、データ提示、対比、等）
- イラスト要素の引き出し
- NGコピー（台本に基づかない盛り禁止）

## 実行手順

### Step 1: ガイドの読み込み

`channel_thumbnail_guide.md` を読み込んで、デザインルールを把握する。

### Step 2: 台本の読み込みと分析

ユーザーが指定した台本ファイルをすべて読み込む。

各台本から以下を抽出する：

| 抽出する情報 | 台本のどこにあるか | 使い方 |
|---|---|---|
| **冒頭フック** | ヘッダー or Block 1冒頭 | → メインキャッチの最有力候補 |
| **コアメッセージ** | ヘッダー | → メインキャッチ or サブテキスト |
| **通説** | 前半で紹介される「常識」 | → 小さく書いて否定する / 対比の片側 |
| **真実** | 後半で明かされる結論 | → メインキャッチ |
| **衝撃データ** | 数字・統計 | → サブテキスト / データ型キャッチ |
| **象徴モチーフ** | テーマから連想 | → 添え物イラスト |

### Step 3: プロンプト生成

各台本につき **3〜6パターン** のプロンプトを生成する。

#### プロンプト構成ルール

各プロンプトは以下の要素を**1行にまとめて**記述する：

```
YouTube動画のサムネイル画像。[狙いたい印象]。背景：[背景の描写]。[位置]：[イラスト要素の描写]。[位置]：[フォント種類] [文字色＋装飾] 「[キャッチコピー]」。[位置]：（サブテキスト）[装飾] 文字「[サブテキスト]」。--ar 16:9 --style raw --q 2
```

#### 1行化のルール

- 元のガイドでは複数行に分けて書いていたプロンプトを **句点（。）で区切って1行に圧縮** する
- 改行は絶対に入れない
- `--ar 16:9 --style raw --q 2` を末尾に必ず付ける
- 各要素間は `。` または ` ` で区切る

#### レイアウトパターンの選択

各台本について、以下の5つのレイアウトパターンから**異なるパターンを3〜6個**選んでプロンプトを作成する：

| パターン | 特徴 | 最適なテーマ |
|---|---|---|
| A. キャラ＋吹き出しドカン型 | 線画キャラ＋巨大吹き出し | 実験・研究者の引用 |
| B. 左右ベタ塗り対比型 | 赤vs青の対比 | 通説vs事実、対比テーマ |
| C. 黒背景テキスト勝負型 | テキスト主役、添え物イラスト | データ系、否定系 |
| D. 図解×マーク破壊型 | 教科書＋赤×マーク | 通説否定、グラフデータ |
| E. テキストサイズ差インパクト型 | 通説小さく→否定巨大 | すべてのテーマ |

### Step 4: 最終出力

すべてのプロンプトを **1つのコードブロック** にまとめて出力する。

#### 出力フォーマット

コードブロックの外に、どのテーマのどのパターンかの説明を書く。
コードブロック内は**プロンプトのみ、1行1プロンプト**。

例（2テーマの場合）：

### ⚠️ 重要な注意事項

1. **1プロンプト = 1行**。絶対に改行しない
2. **コードブロック内にはプロンプトのプレーンテキストのみ**。マークダウン装飾や見出しを入れない
3. **台本に書いていない誇張は禁止**（「全部ウソ」「衝撃の真実」等はNG）
4. **動画タイトルとそのまま同じキャッチコピーにしない**（別角度の表現にする）
5. **メインキャッチは8〜15文字**。長すぎると画像内のテキストが潰れる
6. プロンプトの冒頭は必ず `YouTube動画のサムネイル画像。` で始める
7. 末尾は必ず `--ar 16:9 --style raw --q 2` で終わる
8. 各テーマで**異なるレイアウトパターン**を使い分ける（全部同じ型にしない）

### 使い方（ユーザー向け）

1. このワークフローを実行して、プロンプトがコードブロックに出力される
2. コードブロックの中身をクリップボードにコピー
3. Chromeで Gemini（gemini.google.com）を開く
4. **Gemini Image Auto Generator** 拡張機能のポップアップを開く
5. 「テキスト入力」タブにペースト
6. 「まとめてキューに追加」→「スタート」
7. Geminiが順次プロンプトを処理し、画像を自動生成

## 複数台本の一括処理

### 入力方法

ユーザーは以下のいずれかで台本を指定する：

```
# 方法1: テーマ名を列挙
マシュマロ実験、IQテスト、ドーパミン

# 方法2: パスを指定
台本作成/マシュマロ実験/script.md
台本作成/IQって本当に「頭の良さ」を測れてるの？/script.md

# 方法3: 「全部」「すべて」
台本作成にあるすべての台本
```

### 処理フロー

1. 指定されたすべての台本を読み込む
2. 各台本について Step 2〜3 を実行
3. **すべてのプロンプトを1つのコードブロックにまとめて出力**

> 10本の台本 × 各4パターン = 40プロンプトが1つのコードブロックに出力される
