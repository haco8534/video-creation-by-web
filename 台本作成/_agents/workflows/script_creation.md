# YouTube解説動画 台本作成ワークフロー

---

## 🎯 このワークフローの目的・コンテキスト

### 目指すチャンネルの方向性

**「わかりやすく親しみやすい。でも、ちゃんと本当のことを言っている。」**

一次情報・裏付けのある事実に基づきながら、ずんだもんとめたんの対話によって難しいテーマを誰でも楽しく理解できる形で届けるチャンネル。
AIが生成した台本をそのまま使うのではなく、**リサーチ → 構成設計 → 執筆 → 推敲** の多段階プロセスを経ることで、ワンショット生成では達成できない構成力・ファクトの品質・視聴継続性を実現する。

### なぜ多段階ワークフローなのか

| 問題 | ワンショット生成の限界 | このワークフローの解決策 |
|------|----------------------|------------------------|
| ファクトの正確性 | AIが知識から「それっぽい」内容を生成し、事実誤認が混入しやすい | Step 2で事前にリサーチし、信頼度を付けた上で執筆に使う情報を選別する |
| 構成の品質 | 起承転結が表面的になり、視聴者が離脱しやすい | Step 3を独立したフェーズとして設け、視聴維持率を意識した構成設計を行う |
| キャラクターの一貫性 | 口調がブレたり、両者の役割が曖昧になる | キャラクター定義を明確に固定し、全ブロックで統一する |
| 全体のまとまり | 各セクションが断片的になり、動画としての流れが悪い | Step 5で全体を俯瞰してチェックし、冒頭の問いとまとめの対応を確認する |

### 出力物の最終像

```
d:\myfolder\動画生成\台本作成\
  └── {テーマ名}/
        ├── script.md        ← 完成台本（セクション分け・対話形式）
        ├── research.md      ← 統合キーファクト一覧（信頼度・出典付き）
        ├── research_2a.md   ← チェックポイント：主流・肯定情報（一時ファイル）
        ├── research_2b.md   ← チェックポイント：反論・例外情報（一時ファイル）
        └── research_2c.md   ← チェックポイント：背景・深掘り情報（一時ファイル）
```

## 📋 前提・ルール

- **キャラクター**: ずんだもん（解説役・主体）、めたん（聞き役・質問役）
- **出力形式**: Markdownファイル（`.md`）
- **目標尺**: 約20分（セリフ総量の目安：**6,000〜7,000文字**）
  - 日本語の話速は約300〜350文字/分を基準とする
- **承認ゲート**: **Step 3（構成設計）のみ**。それ以外は自動で次のステップへ進む
- **品質方針**: 親しみやすく分かりやすいが、一次情報・裏付けのある事実に基づく内容を最優先する
- **情報の信頼度基準**:
  - 🟢 **高**: 公式発表・査読済み論文・政府統計・一次情報
  - 🟡 **中**: 信頼性の高いメディア・専門家の発言・二次情報
  - 🔴 **低**: 一般ブログ・未確認情報・出典不明 → **台本には使用しない**

## 📊 視聴維持率を上げるための設計原則

台本執筆・構成設計の全ステップにおいて、以下の原則を常に意識すること。

### 原則1：冒頭30秒が命（ゴールデン30秒）

動画冒頭で視聴者の「見る価値があるか」の判断が下される。
以下のいずれかを必ず冒頭に配置する：

- **驚きのある問いかけ**: 「実は〇〇って、ほとんどの人が誤解しているんです」
- **結論の予告（逆説的フック）**: 「この動画を見終わったら、〇〇に対する見方が180度変わります」
- **共感フック**: 「〇〇って、なんとなくわかってるつもりで実はよくわからない、そんな人のための動画です」

長すぎる挨拶・チャンネル紹介は冒頭に置かない。

### 原則2：「次が気になる」連鎖を作る（未完の法則）

視聴者は「答えがわからない状態」では動画を止めにくい。
各ブロックの**末尾**に、次のブロックへの「問い」や「予告」を置く。

> 例：「じゃあなんで〇〇が起きたのか、それには意外な理由があるんですが……それは次で話しますね」

### 原則3：中盤の中だるみを防ぐ（情報密度のコントロール）

- 20分動画の中盤（8〜14分付近）は最も離脱が起きやすい
- 中盤に**「最も意外性のあるファクト」や「反論・誤解を覆す展開」**を配置する
- 長い説明が続く場合は、めたんの短い質問・リアクションで「息継ぎ」を入れる

### 原則4：段階的な情報開示（Progressive Disclosure）

視聴者が「全部わかった」と思った瞬間に離脱する。
情報を一度に全部伝えず、「実はもう一つあって〜」「ここからが本当の話で〜」のように**段階的に深く掘り下げる**構成にする。

### 原則5：フックとまとめの対応（問いの回収）

冒頭で立てた問いは、**必ずまとめで明示的に回収**する。
これにより視聴者に「最後まで見てよかった」という満足感を与える。

## Step 1 ── テーマ設定 & 核心メッセージの確定

ユーザーからテーマを受け取り、以下を出力する。

1. **コアメッセージ（1文）**: この動画で視聴者に最も伝えるべきことを1文にまとめる
2. **ターゲット視聴者像**: どんな知識レベル・興味を持つ人が見るか
3. **視聴者がよく持つ誤解・前提知識**: 動画の前半で解消すべき誤解を洗い出す
4. **動画の「問い」（フック候補）**: 冒頭で視聴者を引きつける問いかけを3案提案する
5. **視聴者が動画を見終わった後に変わること**: 「視聴前の認識 → 視聴後の認識」を明文化する

完了したら自動的に Step 2 へ進む。

## Step 2 ── マルチ視点リサーチ（3並列 + リーダー統合）

### ⚠️ 並列実行の指示

**Step 2a・Step 2b・Step 2c は同時並列で実行する。** 逐次実行は禁止。

各サブエージェント（Step 2a/2b/2c）は独立してリサーチを行い、それぞれの結果を**チェックポイントファイル**として個別出力する。
全3つのチェックポイントファイルの出力完了後、**リーダーエージェント**（Step 2d）がそれらを統合して最終的な `research.md` を作成する。

```
[Step 2a] 主流・肯定情報のリサーチ ──→ research_2a.md 出力  ─┐
[Step 2b] 反論・例外情報のリサーチ  ──→ research_2b.md 出力  ─┼──→ [Step 2d リーダー] → research.md
[Step 2c] 背景・深掘り情報のリサーチ ─→ research_2c.md 出力  ─┘
```

収集した情報はすべて信頼度（🟢高・🟡中・🔴低）を付けて一覧化する。

### ⚠️ リサーチの量・深さの基準（必ず守ること）

- **各サブステップで最低5回以上のWeb検索**を行う。1回の検索で終わることは絶対に禁止
- **サブステップ全体で最低15件以上のファクト**を収集することを目標とする
- 同じキーワードで繰り返し検索するのではなく、**検索クエリを毎回変えて多角的に調査**する
- 一つのファクトが見つかったら、それを起点に「関連する別の事実・数字・研究」を芋づる式に掘り下げる
- **「情報が少ない」と感じた場合は検索を追加し、十分な量が集まるまでリサーチを継続する**
- リサーチを終了する前に「このテーマについて視聴者が疑問に思いそうなことをすべて答えられるか？」を自問する

### 検索クエリの多様化ガイド

同じ情報源・観点に偏らないよう、以下のパターンでクエリを変えながら検索する：

| クエリパターン | 例（テーマが「〇〇」の場合） |
|---------------|------------------------------|
| テーマ + 研究・論文 | 「〇〇 研究 論文 結果」 |
| テーマ + 統計・データ | 「〇〇 統計 データ 数字」 |
| テーマ + 批判・反論 | 「〇〇 批判 反論 問題点」 |
| テーマ + 歴史・起源 | 「〇〇 歴史 起源 いつから」 |
| テーマ + 誤解・myth | 「〇〇 誤解 嘘 myth」 |
| テーマ + 具体例・事例 | 「〇〇 具体例 事例 実例」 |
| テーマ + 専門家・著者名 | 「〇〇 [著者名] 発言 インタビュー」 |
| テーマ + 最新動向 | 「〇〇 最新 2023 2024 2025」 |
| テーマ + 英語検索 | 英語でのクエリも試みる（学術情報へのアクセス向上） |

### Step 2d ── リーダーエージェント：チェックポイント統合 & `research.md` 生成

**実行タイミング：Step 2a・2b・2c の全チェックポイントファイル（`research_2a.md`, `research_2b.md`, `research_2c.md`）が揃ってから実行する。**

リーダーエージェントは各チェックポイントファイルを**読み込み**、以下の作業を行う：

2. **信頼度の再評価**: 複数の出典から裏付けられたファクトは信頼度を上げる
3. **欠落の補完**: 3つのサブエージェントのカバレッジを比較し、抜けている観点を特定して追加リサーチを行う
4. **優先度付け**: 台本で使用すべき重要ファクトに優先度を付ける
5. **統合ファクト一覧の作成**: 最終的な `research.md` を出力する

## Step 3 ── 構成設計 【⚠️ 承認ゲート】

収集したキーファクトと**視聴維持率の設計原則**をもとに、動画の構成ブロックを設計する。

### 構成設計における必須チェック

構成案を作る前に、以下を確認する：

- [ ] 冒頭30秒以内に強力なフックが置かれているか
- [ ] 各ブロック末尾に次への「引き」があるか
- [ ] 最も意外性のあるファクトは中盤（Block 3〜4付近）に配置されているか
- [ ] 冒頭の「問い」がまとめブロックで明示的に回収されるか
- [ ] 情報が段階的に開示される構成になっているか
- [ ] 各ブロック内の **シーン遷移ポイント**（背景コンテンツの切り替えタイミング）が設計されているか

### 構成案の出力フォーマット

```
【Block N】タイトル（推定時間：〇分 / 目標文字数：〇〇〇〇文字）
  内容：〇〇について解説する
  使用するキーファクト：（Step2で収集したファクトのうち使うもの・信頼度付き）
  担当：ずんだもん（解説）/ めたん（質問・リアクション）
  視聴維持率の仕掛け：（冒頭フック / ブロック末尾の引き / 意外性 / 段階的開示 など）
  視聴者が得るもの：このブロックを見終わった後に視聴者が理解すること
  セクション分類：（導入 / 本論 / トピック〇〇 / 終わりに）
  シーン遷移設計：
    ├─ シーンA（セリフ1〜3）：〈ビジュアルの意図〉
    ├─ シーンB（セリフ4〜7）：〈ビジュアルの意図〉
    └─ シーンC（セリフ8〜10）：〈ビジュアルの意図〉
```

### シーン遷移設計のガイドライン

この台本は、後続の動画生成ワークフローで **背景コンテンツ（比較表・数値表示・フロー図・リスト・引用カードなど）** の切り替えに使われる。
ブロックの中でも話題の転換や情報の種類が変わるポイントで、**シーン（背景コンテンツの塊）を分ける**設計をする。

#### シーン遷移を入れるべきタイミング（積極的に入れること）

| タイミング | 例 |
|-----------|----|
| 話題の内容が切り替わるとき | 「通説の紹介」→「その反論」 |
| 情報の表現形式が変わるとき | 「説明テキスト」→「数値データの提示」 |
| 比較・対照が始まるとき | 「Aの特徴」→「AとBの比較」 |
| 具体例やエビデンスが登場するとき | 「概念の説明」→「研究データの紹介」 |
| まとめ・結論に入るとき | 「議論の展開」→「結論の提示」 |
| 新しい数字・統計が出るとき | 「概要の説明」→「具体的な数値の提示」 |
| 人物やエピソードが変わるとき | 「研究者Aの話」→「研究者Bの話」 |
| セリフが3往復（6行）を超えたとき | 長いシーンを自動的に分割する |

#### 1ブロックあたりのシーン数の目安

| ブロックの推定時間 | 推奨シーン数 |
|-------------------|------------|
| 1〜2分 | 2〜4シーン |
| 2〜3分 | 4〜6シーン |
| 4〜5分 | 6〜10シーン |

**原則：セリフ2〜3往復ごとにシーン遷移マーカーを入れること。** 同じ背景コンテンツが4往復以上続くのは禁止。視聴者が視覚的に飽きないよう、話の内容が少しでも変わったらシーンを切り替える。

### 標準ブロック構成（調整可）

| Block | セクション | 内容 | 推定時間 | 視聴維持率の仕掛け |
|-------|----------|------|----------|-------------------|
| 1 | 導入 | オープニング：フック・問いの提示 | 1〜2分 | ゴールデン30秒フック、動画全体の予告 |
| 2 | 導入 | 前提知識・用語の整理 | 2〜3分 | 「実は〇〇って難しくなかった」という安心感 |

### セクション分けのルール

- **導入（Introduction）**: オープニング〜前提知識の整理まで（視聴者を引き込む役割）
- **本論（Main Content）**: テーマの核心を扱うブロック群
  - 各トピックブロックの見出しは `トピック〇「（内容を示す短いタイトル）について」` とする
  - トピックが複数ある場合は番号を振る（トピック①、トピック②、…）
- **終わりに（Conclusion）**: まとめ・コアメッセージ・視聴者へのメッセージ

### シーン遷移マーカー（`<!-- SCENE: ... -->`）のルール

シーン遷移マーカーは、**動画の背景コンテンツが切り替わるポイント**を台本の中に明示するためのHTMLコメントである。動画生成ワークフローでこの台本を受け取ったエージェントは、このマーカーを基に各シーンのビジュアル設計を行う。

#### フォーマット

```
<!-- SCENE: 〈ビジュアルパターン〉「〈表示内容の概要〉」 -->
```

#### ビジュアルパターン一覧（動画生成側で対応するパターン）

| パターン名 | いつ使うか | 記述例 |
|-----------|-----------|--------|
| タイトルカード | シーンの導入・大見出しを見せるとき | `<!-- SCENE: タイトルカード「1万時間の法則とは？」 -->` |
| 段階的リスト | 項目を1つずつ積み上げて説明するとき | `<!-- SCENE: 段階的リスト「法則の3つのポイント」 -->` |
| 数値インパクト | 衝撃的な数字を大きく表示するとき | `<!-- SCENE: 数値インパクト「たった12%」 -->` |
| 比較対照 | AとBの違いを対比して見せるとき | `<!-- SCENE: 比較対照「通説 vs 実際の研究」 -->` |
| フロー図 | 手順やプロセスを図で見せるとき | `<!-- SCENE: フロー図「意図的練習の4ステップ」 -->` |
| 横並びカード | 2〜3の概念を並べて紹介するとき | `<!-- SCENE: 横並びカード「早期専門化 vs 幅広経験」 -->` |
| 注意喚起 | 重要な補足・警告を目立たせるとき | `<!-- SCENE: 注意喚起「よくある誤解」 -->` |
| 引用カード | 研究者の発言や名言を引用するとき | `<!-- SCENE: 引用カード「エリクソン教授の反論」 -->` |
| テキスト強調 | 一文を大きく表示して印象づけるとき | `<!-- SCENE: テキスト強調「大事なのは量ではなく質」 -->` |
| まとめカード | 本日のポイントを振り返るとき | `<!-- SCENE: まとめカード「今日の3つのポイント」 -->` |

#### 配置ルール

1. **各ブロックの最初のセリフの前**に、必ず1つ目のシーン遷移マーカーを置く
2. **ブロック内で話題や情報形式が変わるポイント**に追加のシーン遷移マーカーを置く
3. シーン遷移マーカーの直後には**空行を1行入れてからセリフを開始する**
4. 1つのシーン遷移マーカーの範囲内にあるセリフ群は、**同じ背景コンテンツの上で再生される**
5. マーカーの `「〈表示内容の概要〉」` 部分には、**画面に表示したい内容の要約**を簡潔に書く（動画生成エージェントへの指示になる）
6. **1つのシーン内のセリフは最大3往復（6行）まで**。それを超える場合は新しいシーン遷移マーカーを入れて分割する
7. **同じ背景が長く続くと動画が単調になる**ため、積極的にシーンを切り替える。「追加のマーカーを入れるか迷ったら、入れる」を原則とする


### セリフの長さ制限（字幕表示のための分割ルール）

台本のセリフは動画内で字幕としても表示される。長すぎるセリフは画面に収まらないため、以下のルールを厳守する。

#### ルール

1. **1セリフの最大文字数：60文字**を目安とする（句読点含む）
2. **60文字を超えるセリフは、意味の切れ目で複数行に分割する**
3. 分割後の各行は、それぞれ独立したセリフ行として `キャラ名：` を付けて記述する
4. 分割の質を維持するため、**文の意味的な区切り**（接続詞「でも」「だから」「そして」の前、句点「。」の後）で分割を推奨
5. 逆に短すぎるセリフ（10文字未満）が連続するのも不自然なため、15〜55文字程度を理想とする

#### ❌ 悪い例（長すぎて画面に収まらない）

```
ずんだもん：実験から約20年後に追跡調査をしたら、4歳のときにマシュマロを我慢できた子どもたちは、我慢できなかった子と比べてSATっていう大学入試の点数が平均210点も高かったのだ。
```

#### ✅ 良い例（分割して字幕に収まる）

```
ずんだもん：実験から約20年後に追跡調査をしたのだ。
ずんだもん：4歳のときにマシュマロを我慢できた子は、できなかった子と比べて……
ずんだもん：SATっていう大学入試の点数が、平均210点も高かったのだ。
```

#### 分割時の注意

- 同じキャラの連続セリフは自然な朗読の「間」として機能する
- 分割後のセリフが不自然にならないよう、各行が単独で意味をなすようにする
- めたんの長いリアクションも同様に分割する

### キャラクターの話し方

**ずんだもん**（解説役）
- 語尾は「〜のだ」「〜なのだ」「〜だぞ」
- 丁寧で分かりやすく、具体例・たとえ話を積極的に使う
- 難しい概念は必ず身近なものに例えて説明する
- 得意げだが押しつけがましくない、親しみやすいトーン

**めたん**（聞き役）
- 視聴者目線で素直に疑問を投げかける
- 「え、それってどういうことですの？」「なんで〜なんですの？」「それ、どういう意味ですの？」
- 難しい言葉が出たら「ちょっと待って、〇〇ってなんですの？」と止める役割
- たまに「それ知ってますわ！〇〇ですわよね？」と言って、ずんだもんに補足・訂正してもらう流れを作る
- 視聴者が「自分も同じことを思ってた」と感じられるような代弁者

### ファクトの扱いルール

- 🟢 信頼度「高」のファクトを優先的に使用する
- 🟡 信頼度「中」は「〜と言われている」「〜とする専門家もいる」など表現をやわらげて使用
- 🔴 信頼度「低」は使用しない
- 数字を使う場合は出所を台本内に自然な形で入れる（例：「〇〇省の調査によると〜」）

### 視聴維持率の仕掛けを台本に組み込む

執筆時に以下を必ず埋め込む：

1. **Block 1冒頭**：Step 3 で決定したフックを使い、最初の30秒以内に「なぜこれを見るべきか」を伝える
2. **各Blockの末尾**：次のブロックへの橋渡しセリフを入れる
   > 例）ずんだもん：「……でも、ここで一つ疑問が出てくるのだ。〇〇がそうなら、なぜ〇〇は〇〇なのか？　実はここに、意外な事実が隠れているのだ」
3. **Block 4（中盤）**：「実は通説と違う」「多くの人が誤解していた」という展開を作り、意外性を最大化する
4. **Block 6末尾**：冒頭の問いを明示的に引用し、それに対する答えを述べて視聴者の「見てよかった」感を作る
